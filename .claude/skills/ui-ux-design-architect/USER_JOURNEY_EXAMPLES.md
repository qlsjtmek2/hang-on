## Hang On 프로젝트 사용자 시나리오

### 시나리오 #1: 처음 앱을 사용하는 사용자

**사용자**: 28세 직장인 지우, 감정 표현에 서툼
**목표**: 앱 설치 후 첫 감정 기록 작성하기
**맥락**: 퇴근 후 집에서, 오늘 힘들었던 일 때문에 기분이 안 좋음

**단계**:
1. 앱 스토어에서 "Hang On" 검색 → 설치
2. 앱 실행 → 온보딩 화면 (3개 화면)
   - 화면 1: "당신의 감정을 날씨로 표현해보세요"
   - 화면 2: "익명으로 누군가와 연결돼요"
   - 화면 3: "시작하기" 버튼
3. 회원가입 화면
   - Google 로그인 선택
   - 권한 허용 (계정 정보)
4. 메인 화면 도착
   - 빈 화면 + "첫 이야기를 시작해보세요" 메시지
   - "털어놓기" 버튼 탭
5. 감정 선택 화면
   - 5단계 날씨 아이콘 표시
   - "비 🌧️" 선택
6. 글쓰기 화면
   - "오늘 팀장님한테 혼났어요. 억울해요."
   - "다 썼어요! 💙" 버튼 탭
7. 공유 옵션 선택 (바텀시트)
   - "지금 나누기" 선택
8. 메인 화면 복귀
   - 첫 기록 카드 표시
   - 성공 토스트: "마음을 나눴어요 💙"

**감정 변화**:
- 시작: 우울함 (-3)
- 중간: 카타르시스 (0)
- 종료: 약간의 위안 (+1)

**주요 touchpoint**:
- 온보딩 화면
- 회원가입 화면
- 메인 화면
- 감정 선택 화면
- 글쓰기 화면
- 공유 옵션 바텀시트

**Pain Points**:
- 온보딩이 너무 길면 지루할 수 있음 → 3개 화면으로 간결하게
- 회원가입 단계가 번거로울 수 있음 → 소셜 로그인 우선 제공

**Opportunities**:
- 첫 기록 작성 완료 시 보상 (뱃지, 포인트 등) → Phase 2
- 온보딩 중 예시 기록 보여주기

### 시나리오 #2: 타인의 기록에 공감하기

**사용자**: 35세 프리랜서 민수, 공감 능력 높음
**목표**: 비슷한 감정을 느끼는 사람에게 위로 전하기
**맥락**: 점심 시간, 커피숍에서 휴대폰 보는 중

**단계**:
1. 앱 실행 → 메인 화면
2. 하단 탭바에서 "누군가와 함께" 탭
3. 타인 기록 피드 화면
   - 감정 유사도 기반 매칭된 기록 20개 표시
   - 스크롤하며 읽기
4. 공감 가는 기록 발견
   - "저도 오늘 힘들었어요. 우리 함께 견뎌요."
   - 카드 내 하트 아이콘 탭
5. 하트 애니메이션 (펄스 효과)
   - "공감을 전했어요" 토스트
6. 메시지 프리셋 선택
   - "함께해요 ✨" 탭
7. 메시지 전송 성공
   - "메시지를 보냈어요" 토스트
8. 다음 기록으로 이동

**감정 변화**:
- 시작: 중립 (0)
- 중간: 공감 (+2)
- 종료: 뿌듯함 (+3)

**주요 touchpoint**:
- 메인 화면 (탭바)
- 타인 기록 피드
- 기록 카드 (하트, 메시지 버튼)
- 메시지 프리셋 바텀시트

**Pain Points**:
- 기록이 너무 많으면 피로할 수 있음 → 일일 20개 제한
- 메시지 프리셋이 마음에 안 들 수 있음 → Phase 2에서 커스텀 메시지

**Opportunities**:
- 공감한 기록 저장 기능 → Phase 2
- 메시지 프리셋 확장 (10개 이상)

### 시나리오 #3: 과거 기록 수정하기

**사용자**: 22세 대학생 유진, 기록을 자주 수정함
**목표**: 어제 쓴 기록에 추가 내용 작성
**맥락**: 밤 11시, 침대에 누워서

**단계**:
1. 앱 실행 → 메인 화면
2. 자신의 기록 목록에서 어제 기록 카드 탭
3. 기록 상세 화면
   - 감정 상태: 비 🌧️
   - 작성 시간: "어제 오후 8:30"
   - 글 내용: "오늘 시험 망쳤어요."
   - 받은 공감: 5개
   - 받은 메시지: 2개 ("힘내세요", "저도 그래요")
4. 우상단 "수정" 버튼 탭
5. 글쓰기 화면 (기존 내용 채워진 상태)
   - 추가 내용 작성: "그런데 친구가 위로해줘서 괜찮아졌어요."
   - "저장" 버튼 탭
6. 기록 상세 화면 복귀
   - 수정된 내용 표시
   - "수정했어요" 토스트

**감정 변화**:
- 시작: 중립 (0)
- 중간: 회상 (-1)
- 종료: 안정 (+1)

**주요 touchpoint**:
- 메인 화면 (기록 목록)
- 기록 상세 화면
- 글쓰기 화면 (수정 모드)

**Pain Points**:
- 수정 후 기존 공감/메시지가 사라질까 걱정 → "기존 공감/메시지는 유지돼요" 안내

**Opportunities**:
- 수정 이력 추적 → Phase 2

---

### 시나리오 #4: 기록 삭제하기

**사용자**: 30세 직장인 현아, 과거 기록 정리 중
**목표**: 오래된 기록 삭제
**맥락**: 주말 오후, 방 청소하면서

**단계**:
1. 앱 실행 → 메인 화면
2. 3개월 전 기록 카드 탭
3. 기록 상세 화면
4. 우상단 "더보기" 메뉴 (⋯) 탭
5. 바텀시트: "수정", "삭제", "공유 설정 변경"
6. "삭제" 선택
7. 확인 다이얼로그
   - "이 기록을 삭제할까요? 받은 공감과 메시지도 함께 사라져요."
   - "삭제" / "취소" 버튼
8. "삭제" 버튼 탭
9. 메인 화면 복귀
   - 기록 목록에서 제거
   - "삭제했어요" 토스트

**감정 변화**:
- 시작: 중립 (0)
- 중간: 고민 (-1)
- 종료: 후련함 (+1)

**주요 touchpoint**:
- 메인 화면
- 기록 상세 화면
- 더보기 메뉴 바텀시트
- 삭제 확인 다이얼로그

**Pain Points**:
- 실수로 삭제할 수 있음 → Soft Delete (30일 복구 가능)

**Opportunities**:
- 휴지통 기능 → Phase 2

## 일반적인 사용자 시나리오 패턴

### 패턴 #1: 온보딩 (Onboarding)

**목표**: 새 사용자에게 앱의 핵심 가치 전달

**구조**:
1. Welcome Screen (환영 메시지)
2. Feature Highlights (주요 기능 3-5개)
3. Permissions Request (권한 요청)
4. CTA (회원가입 또는 시작하기)

**원칙**:
- 최대 3-5개 화면 (너무 길면 건너뜀)
- 시각적 일러스트 사용
- 건너뛰기 버튼 제공

---

### 패턴 #2: 폼 제출 (Form Submission)

**목표**: 사용자 입력 → 검증 → 제출

**구조**:
1. 입력 화면 (필수/선택 표시)
2. 실시간 검증 (에러 메시지)
3. 제출 버튼 (유효성 검사 후 활성화)
4. 로딩 상태 (스피너)
5. 성공/실패 피드백 (토스트 또는 다이얼로그)

**원칙**:
- 긴 폼은 여러 단계로 분할
- 진행 상태 표시 (Progress Bar)

---

### 패턴 #3: 검색 및 필터링 (Search & Filter)

**목표**: 대량의 데이터에서 원하는 항목 찾기

**구조**:
1. 검색창 (상단 고정)
2. 필터 버튼 (정렬, 카테고리)
3. 검색 결과 표시
4. Empty State (결과 없음)

**원칙**:
- 실시간 검색 (타이핑 중)
- 최근 검색어 저장

---

### 패턴 #4: 소셜 인터랙션 (Social Interaction)

**목표**: 사용자 간 상호작용

**구조**:
1. 콘텐츠 표시 (피드, 리스트)
2. 액션 버튼 (좋아요, 댓글, 공유)
3. 실시간 피드백 (애니메이션)
4. 알림 (푸시 또는 인앱)

**원칙**:
- 즉각적인 피드백 (0.5초 이내)
- 애니메이션으로 감정 전달

## 사용자 여정 맵 상세 예시

| 단계 | 행동 | 생각 | 감정 | Touchpoint | Pain Point | Opportunity |
|------|------|------|------|------------|------------|-------------|
| **1. 발견** | 앱 스토어에서 "감정 일기" 검색 | "좋은 앱 있을까?" | 궁금함 (+1) | 앱 스토어 | 리뷰 부족 | ASO 최적화 |
| **2. 설치** | "Hang On" 설치 | "괜찮아 보이네" | 기대감 (+2) | 앱 스토어 | - | - |
| **3. 온보딩** | 3개 화면 읽기 | "익명이라니 좋다!" | 안심 (+3) | 온보딩 화면 | 건너뛰기 유혹 | 간결하게 유지 |
| **4. 회원가입** | Google 로그인 | "빨리 써보고 싶다" | 기대감 (+2) | 가입 화면 | 추가 정보 요구 시 이탈 | 소셜 로그인만 |
| **5. 메인 화면** | 빈 화면 확인 | "뭘 해야 하지?" | 혼란 (-1) | 메인 화면 | 다음 행동 불명확 | "털어놓기" 버튼 강조 |
| **6. 감정 선택** | "비" 아이콘 탭 | "이게 내 기분이야" | 공감 (+2) | 감정 선택 화면 | - | - |
| **7. 글쓰기** | 감정 작성 | "말하니까 시원하네" | 카타르시스 (+3) | 글쓰기 화면 | 500자 제한 부족? | Phase 2 검토 |
| **8. 공유 선택** | "지금 나누기" | "누가 읽을까?" | 기대감 (+2) | 바텀시트 | 옵션 의미 불명확 | 설명 추가 |
| **9. 완료** | 메인 화면 복귀 | "해냈다!" | 뿌듯함 (+4) | 메인 화면 | - | 첫 작성 뱃지 |

### Insights

**주요 발견**:
1. 메인 화면에서 혼란 발생 → CTA 버튼 강조 필요
2. 글쓰기 단계에서 가장 긍정적 → 핵심 경험
3. 공유 선택 시 기대감 → 즉각적인 피드백 중요

**개선 방안**:
1. 메인 화면 빈 상태: "첫 이야기를 시작해보세요" + 큰 "털어놓기" 버튼
2. 공유 옵션: 각 옵션에 설명 추가
3. 첫 작성 완료: 축하 메시지 또는 뱃지
